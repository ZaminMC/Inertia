# Contributing

Inertia is in the foundation phase.

Current priorities:

* version-neutral contracts
* evidence accumulation and traceability
* testable movement-related foundations
* clean behavior-profile boundaries

Contribution rules:

* keep `inertia-api` and `inertia-core` free of server platform code
* prefer direct names and small types
* add tests for engine behavior changes
* do not claim support that the code does not provide yet
* use local references only as research input

If a new type or package makes the project harder to follow, it is probably too early for it.
