# Reference Policy

Inertia uses local anti-cheat references for research only.

That means:

* inspect categories and recurring patterns
* note what data checks usually need
* note common false-positive risks
* model the behavior in Inertia's own neutral API

That does not mean:

* copying source
* pasting snippets
* reusing private names, comments, or config keys
* committing reference repos, decompiled files, or plugin jars

Reference folders stay outside git and outside the shipped project.
